This section explains the runtime ideas behind Devo before you read the
configuration and contributor reference pages.

Devo is model-neutral, but the runtime still needs clear answers to a few
questions:

- Which model profile is active?
- Which provider connection serves that model?
- Which context should be sent to the model?
- Which tools are available, and which require permission?
- Which local extension systems are active?
