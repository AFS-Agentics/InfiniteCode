DeepSeek official API is the recommended first provider for many Devo users.
The recommended Devo setup uses DeepSeek's Anthropic-compatible API path.

Last checked against DeepSeek official docs: 2026-06-14.

DeepSeek's official docs state that the Anthropic-format base URL is:

```text
https://api.deepseek.com/anthropic
```

DeepSeek's current V4 model parameters are `deepseek-v4-pro` and
`deepseek-v4-flash`. Check the official DeepSeek docs before committing config
because provider model names can change.

Official references:

- [DeepSeek API docs](https://api-docs.deepseek.com/)
- [DeepSeek Anthropic API](https://api-docs.deepseek.com/guides/anthropic_api)
- [DeepSeek Models and Pricing](https://api-docs.deepseek.com/quick_start/pricing)
- [DeepSeek change log](https://api-docs.deepseek.com/updates)

## Recommended Provider

Use this provider when you want DeepSeek through the Anthropic Messages wire
API:

```toml
[defaults]
model_binding = "deepseek-v4-pro-deepseek-anthropic"

[providers.deepseek-anthropic]
enabled = true
name = "DeepSeek Anthropic"
base_url = "https://api.deepseek.com/anthropic"
credential = "deepseek_api_key"
wire_apis = ["anthropic_messages"]

[providers.deepseek-anthropic.web_search]
mode = "provider"

[providers.deepseek-anthropic.web_fetch]
mode = "provider"

[model_bindings.deepseek-v4-pro-deepseek-anthropic]
enabled = true
model_slug = "deepseek-v4-pro"
provider = "deepseek-anthropic"
request_model = "deepseek-v4-pro"
display_name = "DeepSeek V4 Pro"
invocation_method = "anthropic_messages"
default_reasoning_effort = "high"
```

Store the API key in `<DEVO_HOME>/auth.json`:

```json
{
  "version": 1,
  "credentials": {
    "deepseek_api_key": {
      "kind": "api_key",
      "value": "sk-your-deepseek-api-key"
    }
  }
}
```

## Flash Binding

Add a second binding if you want both DeepSeek V4 Pro and DeepSeek V4 Flash in
the `/model` picker:

```toml
[model_bindings.deepseek-v4-flash-deepseek-anthropic]
enabled = true
model_slug = "deepseek-v4-flash"
provider = "deepseek-anthropic"
request_model = "deepseek-v4-flash"
display_name = "DeepSeek V4 Flash"
invocation_method = "anthropic_messages"
default_reasoning_effort = "high"
```

Keep `[defaults].model_binding` pointed at whichever binding should start by
default.

## Why Anthropic-Compatible First

Use the Anthropic-compatible provider method for DeepSeek official API because
that path lets Devo use provider-hosted `web_search` and `web_fetch`.

The OpenAI-compatible path can run model requests, but web search commonly
needs separate local or third-party configuration.

## OpenAI-Compatible Alternate

Use this only when you specifically want the OpenAI Chat Completions-compatible
DeepSeek path:

```toml
[defaults]
model_binding = "deepseek-v4-pro-deepseek-openai"

[providers.deepseek-openai]
enabled = true
name = "DeepSeek OpenAI Compatible"
base_url = "https://api.deepseek.com"
credential = "deepseek_api_key"
wire_apis = ["openai_chat_completions"]

[model_bindings.deepseek-v4-pro-deepseek-openai]
enabled = true
model_slug = "deepseek-v4-pro"
provider = "deepseek-openai"
request_model = "deepseek-v4-pro"
display_name = "DeepSeek V4 Pro"
invocation_method = "openai_chat_completions"
default_reasoning_effort = "high"
```

If you need search with this path, configure a local provider:

```toml
[tools.web_search]
mode = "local"
local_provider = "exa"

[tools.web_search.local_providers.exa]
kind = "exa"
credential = "exa_api_key"
max_results = 5
```

Then add `exa_api_key` to `auth.json`.

## Validation Checklist

Run:

```bash
devo doctor
```

Then start Devo and ask a read-only first prompt:

```text
Explain the project structure. Do not edit files yet.
```

If setup fails, check:

- `credential` matches an id in `auth.json`;
- `base_url` matches the selected wire API;
- `wire_apis` includes the binding `invocation_method`;
- `request_model` is listed in current DeepSeek docs;
- the API key has access to the selected model.
