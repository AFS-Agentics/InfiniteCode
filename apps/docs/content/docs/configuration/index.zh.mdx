这里先讲用户视角的配置，再进入 provider 和 wire API 细节。

能用 onboarding 时，先运行：

```bash
devo onboard
```

需要可复现的环境配置、批量预置，或 onboarding 暂时没有覆盖的 provider 形态时，再直接编辑配置文件。

## 阅读顺序

1. [配置基础](./config-basics) 说明 `DEVO_HOME`、用户配置、工作区配置、`auth.json` 和优先级。
2. [提供商配置](./provider-setup) 说明 provider 记录和凭据。
3. [模型绑定](./model-bindings) 说明 Devo 的模型 slug 如何绑定到提供商模型名。
4. [DeepSeek 官方 API](./deepseek-official-api) 给出推荐的 DeepSeek 官方 API 配置。
5. [其他提供商](./other-providers) 给出其他常见提供商形态。
6. [模型解析](./model-resolution) 说明 Devo 如何选择当前模型绑定。
7. [Provider Wire APIs](./provider-wire-apis) 说明支持的 wire API 值。

字段列表见[配置参考](../reference/config-reference)。
