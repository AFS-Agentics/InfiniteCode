InfiniteCode reads user config from `INFINITECODE_HOME` and project config from the active
workspace.

## `INFINITECODE_HOME`

`INFINITECODE_HOME` is InfiniteCode's user config directory.

| Platform | Default `INFINITECODE_HOME` |
| --- | --- |
| macOS/Linux | `~/.infinitecode` |
| Windows | `C:/Users/your-username/.infinitecode` |

If you set `INFINITECODE_HOME`, the path must already exist and must be a directory.
InfiniteCode canonicalizes it before use. If `INFINITECODE_HOME` is unset, InfiniteCode uses the
default path and does not require that directory to already exist.

## Files

| File | Scope | Purpose |
| --- | --- | --- |
| `<INFINITECODE_HOME>/config.toml` | User | Main user config: providers, model bindings, skills, MCP, hooks, tools, logging, updates, and remembered project settings. |
| `<INFINITECODE_HOME>/auth.json` | User | Secret credential values referenced by provider and tool config. |
| `<workspace>/.infinitecode/config.toml` | Project | Workspace-level overrides checked when InfiniteCode is launched with a workspace root. |

Provider API keys should be stored in `auth.json` and referenced by credential
id from `config.toml`.

## Precedence

InfiniteCode merges config in this order:

1. defaults built into InfiniteCode;
2. user config at `<INFINITECODE_HOME>/config.toml`;
3. project config at `<workspace>/.infinitecode/config.toml`;
4. command-line overrides.

Later sources override earlier sources. This means project config can override
user defaults for one repository, and CLI overrides win over both.

Provider config has its own merge behavior for provider and model binding
tables. User config is the usual place for provider credentials and reusable
model bindings. Project config is best for project-specific defaults.

## Credentials

`auth.json` currently stores API-key credentials:

```json
{
  "version": 1,
  "credentials": {
    "deepseek_api_key": {
      "kind": "api_key",
      "value": "sk-your-key"
    }
  }
}
```

Provider config references a credential by id:

```toml
[providers.deepseek-anthropic]
credential = "deepseek_api_key"
```

If a provider references a missing credential id, InfiniteCode fails provider resolution
with an error that names `auth.json`.

## Inspecting Setup

Run:

```bash
infinitecode doctor
```

`infinitecode doctor` prints the config home, config status, provider status, and model
catalog status. Use it after editing config manually.
